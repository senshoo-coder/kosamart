01한 줄 정의

"동네 사장님이 직접 운영하고, 주민이 직접 주문·픽업·배달받는 마이크로 마켓"
대형 플랫폼 수수료 없이 사장님의 마진을 지키고, 동네 단위의 빠른 배송과 매장 픽업을 함께 제공합니다.

02문제 의식

기존 음식·식료품 배달 플랫폼은 다음과 같은 한계가 있습니다.

  • 높은 수수료 부담 — 매출의 10~30%를 수수료로 잃는 동네 가게
  • 복잡한 등록·정산 — 디지털 친화도가 낮은 노포·소상공인의 진입 장벽
  • 광역 배송 모델의 비효율 — 좁은 지역 안의 짧은 배달이 대형 라이더 시스템에 묻힘
  • 그룹 공동구매의 불편 — 카톡 단톡방 운영의 수기 정산·취합 부담
🎯 코사마트는 종로구 한 동네에 집중해, 가게는 가입·관리가 쉬운 사장님 모드를, 주민은 회원가입 1분으로 다회 주문을 가능하게 만들어 이 문제를 해결합니다.

044-Role 시스템

한 플랫폼 안에서 4가지 역할이 상호작용합니다.

🧑

고객

주민. 회원가입 후 동네 가게에서 주문·결제·배달 추적까지.

  • 여러 가게 동시 주문
  • 배달 / 매장 픽업 선택
  • 실시간 상태 추적
🏪

사장님

입점 가게의 운영자. 본인 가게 데이터만 접근 가능 (서버 검증).

  • 주문 입금확인·승인·거부
  • 상품 등록·순서 조정
  • 영업시간·휴무 설정
🚚

배달기사

지역 배달 파트너. 첫 수락 우선 배정 방식.

  • 대기 배달 수락
  • 픽업·완료·실패 처리
  • 배달 사진 첨부
⚙️

관리자

플랫폼 운영자. 전체 가게·사용자·공구를 총괄.

  • 가게 생성·텔레그램 연동
  • 사용자·계정 관리
  • 공구 상품 운영

06차별화 요소

차별 포인트구현 내용
중간 수수료 0%고객→가게 직접 계좌이체 모델. 플랫폼은 수수료를 받지 않음.
동네 단위 운영종로구 5개 동에 한정 — 단거리 배달, 매장 픽업 자유로움
4-Role 자율 운영사장님이 직접 상품·영업시간·순서 관리. 관리자는 가게 생성·정산만.
설치 없음PWA 기반 모바일 웹. 카톡 공유 한 번으로 진입.
실시간 알림텔레그램 봇 통합. 별도 푸시 인프라 없이 즉시 통보.
그룹 공동구매관리자가 공구 캠페인을 운영. 마감 알림·자동 집계 지원.

08데이터 모델 핵심

엔티티설명
users4가지 역할 (customer/owner/driver/admin), 사장님은 store_id로 가게 바인딩
stores가게 정보 (영업시간·휴무·계좌·텔레그램 ChatID)
products가게별 상품 (sort_order 기반 노출 순서 제어)
orders주문 (status: pending → paid → approved → delivering → delivered)
order_items주문 상세 (가격·수량·소계 스냅샷)
deliveries배달 (driver_id, 사진, 상태). 픽업 주문은 생성하지 않음
group_buys공구 캠페인 (관리자 운영)
order_status_logs상태 전이 감사 로그

09보안 & 개인정보 정책

영역정책
인증bcrypt 비밀번호 해시 + httpOnly·Secure·SameSite=lax 쿠키
권한 분리모든 API에서 역할 검증. 사장님은 본인 store_id 매칭 검증 (서버 사이드)
웹훅 보안WEBHOOK_SECRET 헤더 검증으로 외부 위조 차단
입력 검증가격·재고 음수 차단, 파일 업로드 10MB / 이미지 MIME 화이트리스트
PII 자동 파기완료된 주문의 전화번호·주소·닉네임을 일정 기간 후 자동 익명화 (pg_cron)
역할 격리고객은 본인 주문만, 기사는 본인 배달만, 사장님은 본인 가게 주문만 조회
로깅주문 상태 전이는 order_status_logs에 감사 기록
🔒 "필요한 것만, 필요한 사람에게만" — 데이터 최소 노출 원칙을 모든 API 응답에 적용합니다.

10대표 사용자 시나리오

📖 시나리오 A — 평창동 주민 김씨의 저녁 반찬

저녁 7시, 김씨는 카톡으로 받은 코사마트 링크를 열어 가입한다. 반찬가게에서 시금치나물·고등어조림을 담고, 평창동 주소 입력 후 "주문하기". 가게 계좌로 18,500원을 이체하고 닉네임을 입력자명으로 보낸다. 5분 뒤 사장님의 입금확인이 떨어지고, 동네 기사가 수락. 7시 35분, 문 앞에 도착 사진과 함께 배달 완료.

📖 시나리오 B — 빵집 사장님 박씨의 새 메뉴 등록

아침 6시, 박씨는 새로 만든 잡곡식빵을 코사마트에 등록한다. 사진을 한 장 찍어 올리고, 가격 4,500원 입력, 인기 메뉴는 ▲ 버튼으로 위로 올린다. 저장 없이 즉시 반영 — 고객 화면에서도 바로 노출. 첫 주문은 30분 만에 들어왔다.

📖 시나리오 C — 관리자의 단오 공동구매

관리자는 단오 절기에 맞춰 강원도 직배송 단호박 공구를 연다. 마감일을 5/30로 설정하고 텔레그램 그룹에 안내. 30주민이 참여하고, 마감일 새벽에 자동 정산 알림이 발송된다.

11향후 로드맵

단계주요 항목
Phase 1 (현재)4-Role 운영, 종로구 5개 동, 6+ 입점 가게, 텔레그램 알림, 그룹 공구
Phase 2 (단기)매출 정산 리포트 자동화, 단골 리워드, 카드결제 연동 검토
Phase 3 (중기)인근 동(부암·홍지 외) 추가 확장, 기사 인센티브 시스템, 공구 캘린더
Phase 4 (장기)지자체·동네 단체와의 연계, 비영리 공구 모델, 다국어 (외국인 주민) 지원

12한 페이지 요약 (Pitch)

코사마트는 종로구 동네 단위에서 작동하는, 수수료 0%의 마이크로 마켓플레이스입니다.

🏪 사장님은 디지털 진입 장벽 없이 가게를 운영하고,
🧑 주민은 카톡 한 번으로 가입해 여러 가게를 동시에 주문하며,
🚚 배달은 동네 안에서 빠르게,
⚙️ 운영은 단 한 명의 관리자로도 충분합니다.

대형 플랫폼이 놓친 "가까운 곳, 가까운 사람"의 가치를 기술로 다시 잇습니다.
